Compter le nombre de nombres premiers

Auteur : Franck CHAMBON

Il y a une infinité de nombres premiers comme le montre le théorème d'Euclide.

  • 222 est un nombre premier.
  • On suppose que l'ensemble des nombres premiers est fini.
    • Notons NNN le produit de ces nombres.
    • N>1N>1N>1 d'après le point précédent, et donc N+1>2N+1>2N+1>2.
    • Soit ppp le plus petit facteur premier de N+1N+1N+1.
    • Par définition il divise NNN mais ne divise pas 111, donc ne peut pas diviser N+1N+1N+1.
  • Contradiction, il y a une infinité de nombres premiers.

Cependant, il y a un nombre fini de nombres premiers jusqu'à nnn quand nnn est fixé. La répartition des nombres premiers, et sa connaissance fine est en lien direct avec le fameux problème de l'Hypothèse de Riemann.

  • L'article Wikipédia (en anglais) Prime-counting function fournit vite de nombreuses informations à ce sujet.
  • Une vidéo de David Louapre (Science étonnante) en montre quelques aspects.

Montrons différents algorithmes pour calculer le nombre de nombres premiers jusqu'à nnn, pour nnn fixé.

Méthode par force brute

Avec un premier test de primalité

Utilisons la définition d'un nombre premier :

Un nombre est premier si ses seuls diviseurs sont 111 et lui-même.

def est_premier(x):
    "retourne un booléen, True si x est premier, False sinon"
    if x < 2:
        return False
    for d in range(2, x):
        if x % d == 0:
            return False
    return True

def répartition_premiers(n):
    "Retourne le nombre de nombres premiers jusqu'à n"
    return sum(1 for x in range(2, n+1) if est_premier(x))

e = 4
n = 10**e
Qn = répartition_premiers(n)
print(f"Il y a {Qn} nombres premiers jusqu'à 10^{e}.")

Quelle est la complexité de cet algorithme ?

  • La fonction est_premier(x) a un coût en O(x)\mathcal O(x)O(x) ; dans le pire des cas (xxx un nombre premier), la boucle fait presque xxx tours à coûts constants.
  • La fonction répartition_premiers(n) a donc un coût majoré par O(∑x=2n−1x)\mathcal O \left(\sum\limits_{x=2}^{n-1} x\right)O(x=2∑n−1​x), qui est en O(n2)\mathcal O\left(n^2\right)O(n2).
  • Mais en pratique, la boucle fait un tour complet uniquement pour les nombres premiers, dont on sait que la proportion parmi les nombres de 111 à nnn est environ 1ln⁡n\dfrac1{\ln n}lnn1​. Ce qui conduit à la minoration du coût en O(n2ln⁡n)\mathcal O\left(\dfrac{n^2}{\ln n}\right)O(lnnn2​).
  • En conclusion, la complexité est presque quadratique, il est alors possible de l'utiliser raisonnablement jusqu'à n≈104n \approx 10^4n≈104.

En améliorant le test de primalité

Utilisons une propriété d'arithmétique :

Un nombre composé xxx possède un diviseur strict inférieur à x\sqrt xx​.

Ou plutôt sa contraposée :

Si un nombre x>1x>1x>1 ne possède aucun diviseur strict inférieur à x\sqrt xx​, alors xxx est un nombre premier.

Il suffit d'ajouter deux lignes au code précédent :

def est_premier(x):
    "retourne un booléen, True si x est premier, False sinon"
    if x < 2:
        return False
    for d in range(2, x):
        if d * d > x: # ie d > √(x)
            return True
        if x % d == 0:
            return False
    return True

def répartition_premiers(n):
    "Retourne le nombre de nombres premiers judqu'à n"
    return sum(1 for x in range(2, n+1) if est_premier(x))

Pour être plus efficace, il faut pouvoir donner la primalité des nombres par paquets, et non plus de manière individuelle.

Utilisation du crible d'Ératosthène

Le crible d'Ératosthène permet d'obtenir la liste de tous les nombres premiers jusqu'à une certaine limite.

Méthode

  • 000 et 111 ne sont ni premiers, ni composés. Remarque : le statut pour 111 a varié au cours du temps.
  • Un entier supérieur à 111 est soit premier, soit composé.
  • Un entier composé xxx s'écrit d×kd\times kd×k avec d,kd, kd,k entiers et 1<d⩽k<x1 < d\leqslant k < x1<d⩽k<x.

Pendant le crible d'Ératosthène, on marque 000, 111 non premiers, puis tous les nombres composés jusqu'à une certaine limite, ne laissant non marqués que les nombres premiers.

Propriété 1

Si x>1x>1x>1 alors xxx possède au moins un diviseur distinct de 111,
et le plus petit d'entre eux est premier.

Preuve : Soit ppp le plus petit diviseur de xxx distinct de 111, s'il est composé, il s'écrit k×dk\times dk×d avec d,kd, kd,k entiers et 1<d⩽k<x1 < d\leqslant k < x1<d⩽k<x, or ddd est aussi un diviseur distinct de xxx de 111 qui contredit la minimalité de ppp. Ainsi ppp est premier.

Propriété 2

Si x>1x > 1x>1 est composé, soit ppp le plus petit diviseur de xxx distinct de 111,

  • ppp est premier (propriété 1)
  • p2⩽xp^2 \leqslant xp2⩽x

On en déduit que tous les entiers composés inférieurs à xxx possèdent un diviseur premier inférieur à x\sqrt xx​.

Cette propriété est très utile et trop souvent négligée !

Crible en animation, jusqu'à 120


Source : Wikipédia

Une fois les multiples de 222 marqués, on marque les multiples de 333, puis de 555, puis de 777. Il est inutile de marquer les multiples de 111111 en vertu de la propriété 2 ; 112=121>12011^2 = 121 > 120112=121>120.

Quand on commence à marquer les multiples de ppp premier, il suffit de commencer à p2p^2p2, en effet tous les nombres composés inférieurs à p2p^2p2 ont déjà été marqués ; ils possèdent un diviseur premier inférieur à ppp, toujours en vertu de la propriété 2.

Motivation

Cette méthode, le crible d'Ératosthène, possède une bonne complexité, et sa simplicité lui permet de nombreuses variations et optimisations. En pratique cette méthode est plus rapide que d'autres cribles, plus délicats à implémenter, ayant pourtant une meilleure complexité théorique.
Cette méthode est aussi une base de réflexion pour construire d'autres cribles : on peut parfois construire globalement un tableau de valeurs d'une fonction sur un intervalle, plutôt que de calculer individuellement chaque valeur.

Implémentation

Comprendre le code proposé ci-dessous

  • On donne la docstring.
  • On initialise une liste de booléens (crible), on fait comme si tous les entiers étaient premiers (True).
  • On marque 000 et 111 comme non premiers (False).
  • On fait une boucle pour ppp partant de 222, en incrémentant ppp de 111 à chaque tour :
    • si ppp est premier, alors on va marquer tous les entiers kkk multiples de ppp à partir de 2p2p2p comme non premiers.
    • range(p*p, n+1, p) donne exactement les multiples de p allant de p2p^2p2 inclus, jusqu'à nnn inclus.
  • Quand p2>np^2 > np2>n, on est sûr d'avoir marqué tous les composés ; d'après la propriété 2.

Une proposition en Python

def est_premier(x):
    "retourne un booléen, True si x est premier, False sinon"
    if x < 2:
        return False
    for d in range(2, x):
        if d * d > x: # ie d > √(x)
            return True
        if x % d == 0:
            return False
    return True

def Ératosthène(n):
    """
    Retourne le crible des nombres premiers inférieurs à n.
    crible[x] vaut True ou False, suivant que x est premier ou non.
    """
    if n < 0:
        return []
    crible = [True] * (n+1)
    crible[0] = False
    if n >= 1:
        crible[1] = False
    p = 2
    while p * p <= n:
        if crible[p]:
            for k in range(p * p, n+1, p):
                crible[k] = False
        p += 1
    return crible

# TEST
for n in range(1000):
    crible = Ératosthène(n)
    for x in range(1, n+1):
        assert crible[x] == est_premier(x), f"Échec du test pour borne = {n}, et x = {x}"
print("Test réussi")
print()

n = 100 # à modifier un peu si vous voulez
print(f"La liste des nombres premiers jusqu'à {n} :")
crible = Ératosthène(n)
print([p for p in range(n) if crible[p]])
print()

Complexité

Complexité du crible d'Ératosthène

Uniquement compréhensible en post-bac

Le nombre d'opérations élémentaires pour faire ce crible pour les entiers inférieurs à nnn est :
∑p=2nn−p2pln⁡p∼∫2nn−x2xln⁡xdx∼nln⁡(ln⁡n)\sum_{p=2}^{\sqrt n} \frac {n-p^2} {p\ln p} \sim \int_{2}^{\sqrt n} \frac {n-x^2}{x\ln x} \mathrm{d}x \sim n\ln(\ln n)p=2∑n​​plnpn−p2​∼∫2n​​xlnxn−x2​dx∼nln(lnn)

Explication : avec la probabilité ln⁡p\ln plnp, ppp est premier, et dans ce cas, on raye des nombres de p2p^2p2 à nnn, par pas de ppp. Ensuite on obtient un équivalent sous forme d'intégrale, et le résultat attendu.

La complexité est :

  • quasi-linéaire en temps : Θ(nln⁡(ln⁡n))\Theta\left(n \ln(\ln n)\right)Θ(nln(lnn)),
  • et linéaire en espace.

Complexité pour le compte de nombres premiers

Faire le compte à la fin est plus rapide que le crible, ainsi on obtient une complexité finale en Θ(nln⁡(ln⁡n))\Theta\left(n \ln(\ln n)\right)Θ(nln(lnn)) avec cette méthode.

En pratique, on a pu compter le nombre de nombre premiers jusqu'à 10710^7107 avec ce code simple en Python et en quelques secondes. Il est possible de l'améliorer un peu :

  • En travaillant par tranches pour utiliser moins de mémoire, et donc rester dans la mémoire cache de l'ordinateur ; calculs bien plus rapides. On parle de crible par segments.
  • En utilisant une autre structure de données que la liste d'entiers Python. Il vaudra mieux utiliser les bytearray.
  • En ne considérant que 222, puis les nombres impairs, on gagne un facteur 222 en espace et en temps.
  • En mettant à part 222 et 333, les nombres premiers sont de la forme 6k±16k±16k±1, où k∈N∗k\in\mathbb N^*k∈N∗. Ce qui peut faire gagner encore un peu de temps et d'espace.
  • On peut continuer un peu ainsi ; avec 222, 333 et 555 à part, les nombres premiers sont congrus à ±1±1±1, ±7±7±7, ±11±11±11, ou ±13±13±13 modulo 303030. Ces considérations s'avèrent contre-productives en Python, où un code avec un style plus fonctionnel sera plus rapide qu'un code élaboré au style impératif. La raison étant que les parties fonctionnelles sont traitées avec des fonctions bien plus rapides.

On peut alors, avec Python, sans trop de mal, aller jusqu'à n=109n = 10^9n=109 en moins d'une minute.

Peut-on faire nettement mieux ?

L'algorithme de Meissel-Lehmer

  • Ernst Meissel (1826-1895) était un astronome et mathématicien allemand.
  • Derrick Lehmer (1905-1991) était un mathématicien américain.

Seul le compte total des nombres premiers nous intéresse, pas la liste des nombres premiers, ni même un tableau de booléens. En s'inspirant du crible d'Ératosthène, on va compter le nombre de nombres non rayés, sans en faire la liste !

La méthode initiée par Meissel, simplifiée par Lehmer en 1959, a été ensuite améliorée en 1985 par Lagarias, Millerand et Odlyzko. Un papier de M. Deleglise et J. Rivat en fait l'écho. Le compte des nombres premiers jusqu'à 101810^{18}1018 a pu alors être effectué. En 2001, Xavier Gourdon a encore amélioré la méthode pour atteindre 102210^{22}1022. Actuellement les travaux de Kim Walish ont permis d'atteindre un record à 102710^{27}1027.

La méthode

Notons Q(n,B)Q(n, B)Q(n,B) le nombre d'entiers dans [ ⁣[2..n] ⁣][\![2..n]\!][[2..n]] qui restent après avoir rayé tous les multiples kpkpkp pour p⩽Bp\leqslant Bp⩽B, et k>1k>1k>1.

Exemples :

  • Q(100,1)=99Q(100, 1) = 99Q(100,1)=99 ; seuls 000 et 111 sont rayés.
  • Q(100,2)=50Q(100, 2) = 50Q(100,2)=50 ; il reste 222 et 3,5,7,9,11,⋯993, 5, 7, 9, 11, \cdots 993,5,7,9,11,⋯99.
  • Q(100,10)=25Q(100, 10) = 25Q(100,10)=25; il ne reste que les nombres premiers inférieurs à 100100100.

Comment calculer Q(n,B)Q(n, B)Q(n,B) ?

  • Q(n,1)=n−1Q(n, 1) = n-1Q(n,1)=n−1 ; aucun nombre n'a été rayé.
  • Si BBB est composé, alors Q(n,B)=Q(n,B−1)Q(n, B) = Q(n, B-1)Q(n,B)=Q(n,B−1) ; on ne raye rien.
  • Si B2>nB^2 > nB2>n, alors Q(n,B)=Q(n,B−1)Q(n, B) = Q(n, B-1)Q(n,B)=Q(n,B−1) ; tous les composés inférieurs à nnn ont déjà été rayés. Il ne reste que des nombres premiers.

Il reste donc uniquement à déterminer Q(n,p)Q(n, p)Q(n,p), pour ppp premier inférieur ou égal à n\sqrt nn​.

Quels sont les nombres rayés associés au nombre premier ppp ?

Il s'agit des nombres p×kp\times kp×k où k∈[ ⁣[2..n/p] ⁣]k\in [\![2..n/p]\!]k∈[[2..n/p]] avec kkk dont le seul diviseur inférieur à ppp est 111.

Exemple :

  • À l'étape p=5p=5p=5, on raye les nombres : 5×5,5×7,5×11,5×13,5×17,⋯ ,5×25,⋯ ,5×35,5×37,⋯5\times 5,5\times 7,5\times 11, 5\times 13, 5\times 17,\cdots, 5\times 25,\cdots,5\times 35,5\times 37,\cdots5×5,5×7,5×11,5×13,5×17,⋯,5×25,⋯,5×35,5×37,⋯
  • Il s'agit de rayer le produit de 555 par : 5,7,11,13,17,⋯ ,25,⋯ ,35,37,⋯5, 7, 11, 13, 17,\cdots, 25,\cdots, 35, 37,\cdots5,7,11,13,17,⋯,25,⋯,35,37,⋯

De manière générale :

  • La liste précédente est l'ensemble des nombres non rayés avant l'étape ppp, que l'on limite à n/pn/pn/p, et sans compter les nombres premiers inférieurs à ppp.
  • La quantité de nombre premiers inférieurs à ppp est Q(p−1,p−1)Q(p-1, p-1)Q(p−1,p−1).
  • La quantité de nombres non rayés avant l'étape ppp, limité à n/pn/pn/p, est Q(n/p,p−1)Q(n/p, p-1)Q(n/p,p−1).

On en déduit que pour ppp premier inférieur ou égal à n\sqrt{n}n​ :
Q(n,p)=Q(n,p−1)−(Q(n/p,p−1)−Q(p−1,p−1))Q(n, p) = Q(n, p-1) - (Q(n/p, p-1) - Q(p-1, p-1))Q(n,p)=Q(n,p−1)−(Q(n/p,p−1)−Q(p−1,p−1))

On rappelle que si ppp n'est pas premier, on a : Q(n,p)=Q(n,p−1)Q(n, p) = Q(n, p-1)Q(n,p)=Q(n,p−1) pour tout nnn.

Ceci nous pousse naturellement à utiliser de la programmation dynamique. On garde un ensemble de valeurs Q(m,p−1)Q(m, p-1)Q(m,p−1) pour mmm variés. On en déduira un ensemble de valeurs Q(m,p)Q(m, p)Q(m,p) pour les mêmes mmm.

Les valeurs de mmm nécessaires sont les nombres kkk inférieurs ou égaux à n\sqrt nn​ et les quotients entiers n/kn/kn/k toujours pour kkk inférieur ou égal à n\sqrt nn​.

On stocke ces valeurs dans un dictionnaire.

Implémentation

def est_premier(x):
    "retourne un booléen, True si x est premier, False sinon"
    if x < 2:
        return False
    for d in range(2, x):
        if x % d == 0:
            return False
    return True

def répartition_premiers(n):
    "Retourne le nombre de nombres premiers inférieurs à n"
    if n < 2:
        return 0
    r = int(n**0.5) # partie entière de racine carrée de n
    m_lst = [n//k for k in range(1, r+1)]
    m_lst += [k for k in range(m_lst[-1]-1, 0, -1)]
    p = 1
    Q = {m: m-1 for m in m_lst}
    for p in range(2, r+1):
        if Q[p-1] < Q[p]: # p est premier
            qp = Q[p-1] # nombre de premiers inférieurs à p
            p2 = p*p
            for m in m_lst:
                if m < p2:
                    break
                Q[m] -= Q[m//p] - qp
    return Q[n]

#TEST
for n in range(100):
    assert répartition_premiers(n) == \
        sum(1 for x in range(2, n+1) if est_premier(x)), n
print("Tests réussis")
